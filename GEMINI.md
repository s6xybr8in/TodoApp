# TodoApp 개발 로드맵

@README.md에 명시된 요구사항을 기반으로 다음과 같은 개발 로드맵을 제안합니다.

---

## Phase 1: 기본 투두리스트 기능 구현

**목표:** 중요도, 진행률, 기간을 포함하는 기본적인 투두리스트의 핵심 기능을 구현합니다.

*   **1-1. 데이터 모델링:**
    - [ O ]*   `Todo` 데이터 모델(클래스)을 생성합니다. (ID, 내용, 중요도, 진행률, 시작일, 종료일 포함)
*   **1-2. UI/UX 구현:**
    - [ O ]   투두리스트를 보여주는 메인 화면을 구성합니다.
    - [ O ]   새로운 투두를 추가하거나 기존 투두를 수정하는 화면을 구현합니다.
    - [ O ]   중요도(예: 상/중/하), 진행률(예: 슬라이더), 날짜(Date Picker)를 선택할 수 있는 UI 컴포넌트를 적용합니다.
*   **1-3. 로컬 데이터베이스 연동:**
    * - [ O ]   `sqflite` 또는 `hive`와 같은 로컬 데이터베이스를 사용하여 투두 데이터를 기기에 저장, 조회, 수정, 삭제(CRUD)하는 기능을 구현합니다.

## Phase 2: 캘린더 뷰 기능 구현

**목표:** 투두 항목들을 월별 캘린더 뷰에서 확인하고, 날짜별 달성도를 시각적으로 표시합니다.

*   **2-1. 캘린더 UI 구현:**
    *   `table_calendar`와 같은 라이브러리를 사용하여 캘린더 UI를 구현합니다.
*   **2-2. 데이터 연동:**
    *   투두 항목의 시작일과 종료일을 기준으로 캘린더에 이벤트를 표시합니다.
*   **2-3. 달성도 표시:**
    *   완료된 투두가 있는 날짜에 특별한 마커나 색상으로 달성 여부를 시각적으로 표시합니다.

---

## Phase 3: 외부 캘린더 연동 (심화)

**목표:** 구글 또는 네이버 캘린더와 연동하여 투두 항목을 동기화합니다.

*   **3-1. API 연동 연구 및 구현:**
    *   구글 캘린더 API 또는 네이버 캘린더 API 사용법을 리서치합니다.
    *   OAuth 2.0 인증을 통해 사용자의 캘린더에 접근하고 동기화하는 로직을 구현합니다. (복잡성이 높으므로 핵심 기능 완성 후 진행 권장)

---

## Phase 4: 홈 위젯 기능 구현

**목표:** 앱에 들어가지 않고도 홈 화면에서 간단한 투두리스트를 확인할 수 있는 위젯을 제공합니다.

*   **4-1. 네이티브 위젯 구현:**
    *   Android(Kotlin/Java) 및 iOS(Swift) 플랫폼별로 홈 위젯을 구현합니다.
*   **4-2. 데이터 공유 및 업데이트:**
    *   `home_widget`과 같은 패키지를 사용하여 Flutter 앱과 홈 위젯 간에 투두 데이터를 공유하고, 주기적으로 위젯의 내용을 업데이트하는 로직을 구현합니다.

---

## Phase 5: 테스트 및 고도화

**목표:** 애플리케이션의 안정성을 확보하고 사용자 경험을 개선합니다.

*   **5-1. 테스트 코드 작성:**
    *   각 기능(데이터 모델, 비즈니스 로직, UI)에 대한 단위 테스트 및 위젯 테스트를 작성하여 안정성을 검증합니다.
*   **5-2. UI/UX 개선:**
    *   전체적인 디자인을 다듬고 사용자 피드백을 반영하여 사용성을 개선합니다.
*   **5-3. 추가 기능 (선택):**
    *   마감일 알림, 테마 변경 등 부가적인 기능을 추가하여 앱의 완성도를 높입니다.
